# Security policy

## Reporting a vulnerability

Please do not publish security vulnerabilities in a public issue.

Report a suspected vulnerability privately through GitHub's **Security** tab and
the repository's private vulnerability reporting feature when available.

Include:

- the affected page or file;
- steps to reproduce the problem;
- the possible impact;
- screenshots or a minimal example, without including passwords or API keys.

## Secrets

API keys, passwords, recovery codes and private tokens must never be committed to
this repository or pasted into an issue. Runtime secrets belong in the hosting
provider's encrypted secret storage.

## Supported version

The current version on the `main` branch and the deployed GitHub Pages site are
supported. Older installed copies should be refreshed before testing a report.
